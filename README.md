# FinanceTracker

FinanceTracker is a single-user personal finance tracker for managing accounts,
transactions, categories, recurring transactions, budgets, and dashboard
summaries.

## Tech Stack

- Backend: FastAPI, SQLAlchemy, Alembic, PostgreSQL
- Frontend: Vue 3, TypeScript, Vite, PrimeVue, Pinia
- Auth: one application password from the environment, with JWT bearer tokens
- Runtime: Docker Compose with PostgreSQL in a named Docker volume

## Features

- Password login for a single user
- Account management with computed balances
- Income and expense categories seeded on startup
- Transaction tracking with filters
- Recurring transaction generation
- Monthly and annual budgets by account or category
- Dashboard summary with balances, cash flow, budget progress, and recent transactions

## Quick Start

Start the full application with Docker Compose:

```bash
docker compose up -d
```

The stack exposes:

- Frontend: http://localhost:5173
- Backend API: http://localhost:8001
- API docs: http://localhost:8001/docs
- Health check: http://localhost:8001/health
- PostgreSQL: localhost:5432

The backend container waits for PostgreSQL, runs `alembic upgrade head`, seeds
default categories during FastAPI startup, and then starts Uvicorn.

Use the password from `APP_PASSWORD` in `backend/.env` to log in.

## Configuration

Docker Compose loads service environment from `backend/.env`. If the file does
not exist yet, create it from the example file before starting the stack:

```bash
cp backend/.env.example backend/.env
```

Important variables are:

- `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `POSTGRES_PORT`
- `BACKEND_PORT`, `FRONTEND_PORT`, `VITE_API_BASE_URL`
- `APP_PASSWORD`, `SECRET_KEY`, `DATABASE_URL`

Inside Docker, `DATABASE_URL` must use the database host name `db`.
For backend runs outside Docker, use a database URL that points to the host
database, for example `localhost`.

Port mappings such as `BACKEND_PORT`, `FRONTEND_PORT`, and `POSTGRES_PORT` are
Compose interpolation values. To override those, export matching variables in
your shell or define them in a root-level Compose `.env` file.

Local secrets should go in `backend/.env` and should not be committed.

## Common Commands

Start or restart the stack:

```bash
docker compose up -d
```

Rebuild containers after dependency or Dockerfile changes:

```bash
docker compose up -d --build
```

View logs:

```bash
docker compose logs -f backend
docker compose logs -f frontend
```

Run backend tests in the backend container:

```bash
docker compose exec backend pytest
```

Stop containers without deleting database data:

```bash
docker compose stop
```

Do not run `docker compose down -v` unless you intentionally want to delete the
PostgreSQL volume and all local app data.

## Local Development Without Docker

Backend:

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --reload --port 8001
```

Frontend:

```bash
cd frontend
npm install
npm run dev
```

Set `VITE_API_BASE_URL` to the backend URL if it differs from
`http://localhost:8001`.

## API Overview

Authentication:

- `POST /auth/login` with `{ "password": "..." }`

Protected resources:

- `/accounts`
- `/categories`
- `/transactions`
- `/recurring`
- `/budgets`
- `/dashboard/summary`

All protected routes require an `Authorization: Bearer <token>` header.

## Data Notes

- Account balances are computed from opening balance and transactions.
- Budget progress is computed from transactions at query time.
- Recurring transactions are generated by `backend/app/services/recurring.py`.
- Account-to-account transfers and live exchange-rate conversion are not part
  of the current scope.
