Docker
======
    docker compose up -d              # start everything
    docker compose down               # stop (data volume persists)
    docker compose down -v            # stop + wipe DB volume
    docker compose logs -f backend    # tail backend logs

Database (PostgreSQL)
=====================

Each time we add a model (accounts, transactions, budgets, etc.), the workflow will be:

  # 1. Write the SQLAlchemy model in app/models/
  # 2. Generate the migration
  docker compose exec backend alembic revision --autogenerate -m "add accounts table"
  # 3. Apply it
  docker compose exec backend alembic upgrade head

  Alembic commands:

  # Generate a new migration
  docker compose exec backend alembic revision --autogenerate -m "add categories table"

  # apply it
  docker exec -it financetracker-backend-1 alembic upgrade head

  # Roll back the last migration
  docker compose exec backend alembic downgrade -1

  # See current migration status
  docker compose exec backend alembic current


Access pgsql shell
==================
docker exec -it financetracker-db-1 psql -U financeuser -d financedb

  Once inside, useful commands:

  -- list all tables
  \dt

  -- check accounts table structure
  \d accounts

  -- see all data
  SELECT * FROM accounts;

  -- exit
  \q
